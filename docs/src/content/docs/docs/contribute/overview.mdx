Want your workshop launchable in Orchestra? The journey is the same whether you
start from a Bioconductor workshop package or an image you already have:

<Steps>

1. **Get a container.** Either build a Bioconductor workshop package (which
   produces one for you) or bring an image you already publish.

2. **Make the image Orchestra-ready.** Orchestra fronts every session with its
   own sign-in, so the app inside must start unauthenticated on a single port.

3. **Author a template.** A small YAML file describes one catalog entry — image,
   port, resources, duration.

4. **Submit it.** Fill in the issue form; a bot opens the PR after a maintainer
   approves. (Comfortable with YAML? Hand-editing a template file and opening the
   PR yourself is the fallback.) It appears in the catalog on the next deploy.

</Steps>
