# GitHub Copilot Instructions for Orchestra Frontend

## Development Workflow

### Always Use Small Commits & PRs
- Feature branch for each UI change
- Atomic commits for components, hooks, pages
- Test changes before committing
- Update types when API changes

### Frontend-Specific Workflow
```
feature/workshop-status-display
feature/create-workshop-form
fix/type-safety-workshop-card
docs/component-documentation
```

### Type Safety First
- Always use generated types from API
- Run `just generate-types` after API updates
- Prefer type-safe API calls
- Handle loading/error states properly

### Component Development
- One component per commit when reasonable
- Include stories/tests for new components
- Update existing components atomically
- Consider accessibility in changes

### Before Frontend Changes
1. Ensure API types are current: `just update-types`
2. Create feature branch
3. Plan component hierarchy changes

### Quality Standards
- Run `just quality` (lint, type-check, test)
- Test in browser before committing
- Consider mobile responsiveness
- Follow existing patterns
